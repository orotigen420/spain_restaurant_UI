# レイアウト調整およびSass（SCSS）導入ログ (2026-06-29)

本日実施した、iPad横向き（想定サイズ: 1516×1024）向けレイアウトのスクロール問題の解消、およびSass（SCSS）の導入手順と仕組みについてのまとめです。

---

## 1. 課題と解決アプローチ

### 課題
* iPadの横向きを想定し Figma（1516×1024）で設計していたが、ブラウザで表示した際に若干の縦スクロールが発生してしまっていた。
* レスポンシブ対応のために、毎回複雑な計算（`calc` など）を個別のCSSに書くのが煩雑であった。

### 解決アプローチ
1. **ビューポート高さを基準にしたアスペクト比固定レイアウト**
   * `html, body` のスクロールを禁止し、全体を中央寄せに配置。
   * ルート要素（`#root`）の高さに `100dvh`（動的ビューポート高さ）を指定し、`aspect-ratio: 1516 / 1024` で横幅を自動決定する。
2. **Sass（SCSS）の導入と `rem()` 関数による一括スケーリング**
   * `html` の `font-size` を画面の高さに合わせて可変にする（`1024px` の高さのときに `16px` になるように設定）。
   * Sassの `rem()` 関数を自作し、Figmaのピクセル値を渡すだけで自動的に適切な `rem` に変換・伸縮されるようにする。

---

## 2. システムの仕組み（Vite + Sass 自動注入）

通常、Sassの関数を使うには各SCSSファイルで手動インポートが必要ですが、`vite.config.ts` の設定により自動化しています。

* **`loadPaths: ['src']`**: Sassに対して `src` ディレクトリを基準にファイルを探索するよう指示。
* **`additionalData`**: 各SCSSファイルのビルド時に、先頭に自動で `@use "styles/functions" as *;` を挿入。(`_functions.scss` 自体が無限ループしないように除外判定付き)

これにより、各コンポーネントのCSS（SCSS）ファイルでは、何もインポートせずにいきなり `rem(数値)` を使用できます。

---

## 3. 実際の実装とコードの変更点

### ① 共通関数の作成
新規作成したファイルで、Figmaの `px` 値を `rem` に変換する関数を定義しています。
* **ファイル**: `src/styles/_functions.scss`
```scss
@function rem($px){ 
    @return calc(#{$px}rem/16);
}
```

### ② Viteの設定変更
Sassのインポート解決と自動注入の設定を行いました。
* **ファイル**: `vite.config.ts`
```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  css:{
    preprocessorOptions:{
      scss:{
        additionalData: (content, filename) => {
          if (filename.includes('_functions.scss')) {
            return content;
          }
          return `@use "styles/functions" as *;\n${content}`;
        },
        loadPaths: ['src']
      }
    }
  }
})
```

### ③ 全体レイアウトと基準フォントサイズの設定
`index.css` を `index.scss` にリネームし、レイアウトと `rem` の基準値を設定しました。
* **ファイル**: `src/index.scss`
```scss
html, body {
  width: 100%;
  height: 100%;
  background: #000;
  overflow: hidden;
  display: flex;
  justify-content: center;
  align-items: center;
  font-size: calc((16 / 1024) * 100dvh); /* 1024px時に16pxになる基準値 */
}

#root {
  height: 100dvh;
  aspect-ratio: 1516 / 1024;
  max-width: 100vw;
  max-height: 100dvh;
  width: auto;
}
```
*(※ `src/main.tsx` のインポートも `index.scss` に変更済み)*

### ④ コンポーネントへの適用例
CSS ModulesファイルをSCSSにリネームし、`rem()` 関数を適用しました。
* **ファイル**: `src/component/screens/FirstView/style.module.scss`
```scss
.info-holder {
  position: absolute;
  z-index: 2;
  top: rem(140);
  left: rem(170);
  width: rem(550);
  display: flex;
  justify-content: flex-start;
  flex-direction: column;
}

h1 {
  font-size: rem(150);
}
```
*(※ `FirstView/index.tsx` のインポートも `style.module.scss` に変更済み)*
